OpenClaw

OpenClaw Skill 快速安装

Skill已上架ClawHub，可直接安装。

Install Skill 
clawhub install nitan

Skill安装完后，通知OpenClaw: "我已经安装nitan skill，帮我查看一下论坛今日热帖"；

如需登录，可以私聊OpenClaw你的账户密码或者自己修改openclaw.json配置文件。

Optinal: Config Nitan Account 

"nitan": {
  "enabled": true,
  "env": {
    "NITAN_USERNAME": "xxxyyy",
    "NITAN_PASSWORD": "你的密码"
  }
}

Quickstart

按环境快速接入

根据自己的平台查看对应安装方式。

  1. macOS 或 Linux 本地安装 Node.js 18+。
  2. 直接运行 npx -y @nitansde/mcp@latest 启动服务。
  3. 建议再运行一次 doctor 做环境检查。

Optional: 让Agent使用需要安装nitan Skill

Optional login: 启动前设置 NITAN_USERNAME 与 NITAN_PASSWORD。不设置时可以公共只读访问。

Local Run 
npx -y @nitansde/mcp@latest
npx -y @nitansde/mcp@latest doctor
  1. 打开 Claude Desktop 配置文件。
  2. 把下面的 mcpServers.nitan 配置写入并保存。
  3. 重启 Claude Desktop，确认 connector 里出现 nitan。

配置文件路径：macOS 为 ~/Library/Application Support/Claude/claude_desktop_config.json，Windows 为 %APPDATA%\Claude\claude_desktop_config.json。

Optional login: 在配置里添加 env 字段并填入 NITAN_USERNAME / NITAN_PASSWORD。

Claude Config 
{
  "mcpServers": {
    "nitan": {
      "command": "npx",
      "args": ["-y", "@nitansde/mcp@latest"]
    }
  }
}
  1. 先部署 MCPHub（Docker 镜像：samanhappy/mcphub）。
  2. 进入 MCPHub 仪表盘，新建一个 STDIO server。
  3. Command 填 npx，Arguments 填 -y @nitansde/mcp@latest，保存后会自动安装并启动。
  4. 对外访问通常使用 /mcp/nitan endpoint。

这套方式适用于 Linux Server、NAS 和 GCP 等远程主机。建议修改 MCPHub 默认管理员凭据并开启 Bearer Authentication。

Optional login: 在 MCPHub 的 Environment Variables 里添加 NITAN_USERNAME / NITAN_PASSWORD（可选再加 TIMEZONE）。

MCPHub Setup 
# Start MCPHub
docker run -p 3000:3000 samanhappy/mcphub

# Add Nitan MCP in MCPHub
Server Name: nitan
Server Type: STDIO
Command: npx
Arguments: -y @nitansde/mcp@latest

Environment Variables:
NITAN_USERNAME=your_id
NITAN_PASSWORD=your_password
TIMEZONE=America/Los_Angeles

FAQ

先看这些，再开始排障

先看这些高频问题；更完整的说明见 README 和论坛贴。

为什么官方 Discourse MCP 不够，需要单独做 Nitan MCP？
官方版本偏通用，对美卡论坛兼容较差。Nitan MCP 在它的基础上补了登录、通知和更多论坛专用工具。
不登录也能用吗？
能用，但会少很多能力：私有板块、通知，以及部分高频访问场景。若账号开启二步验证，请先关闭。
为什么会出现 403 / Cloudflare challenge？
Cloudflare会对bot流量进行阻拦。如果失败，会提示这个。在 macOS 支持了浏览器 fallback，无法访问论坛时会弹出浏览器来抓取页面。
云服务器是不是更容易被 Cloudflare / CSRF 卡住？
是。云 IP 更容易遇到 403 / CSRF，尤其是Google Cloud；本地电脑、NAS 或家宽网络通常更稳。
not_logged_in 导致通知不可用，如何处理？
通知属于登录态能力。配置 NITAN_USERNAME 与 NITAN_PASSWORD，或通过 interactive login 完成登录。如果使用Skill，可以询问Agent如何配置。
macOS 报 Playwright missing 怎么修复？
执行 npm install --no-save playwright，再执行 npx playwright install chromium，最后运行 doctor 验证。
Python curl_cffi script produced no output! 是什么意思？
通常是 Python 路径或依赖有问题。若依赖装在 conda / venv 里，额外传 --python_path /path/to/python；必要时补装 cloudscraper 和 brotli。
spawn npx ENOENT 是什么意思？
这通常不是项目逻辑问题，而是本机没有正确安装 Node.js / npm，或者命令没进 PATH。先确认 npx 在命令行里能直接执行，再回到客户端重新加载配置。
profile lock / SingletonLock 如何恢复？
当前版本支持锁恢复：检测到锁冲突后会终止对应 profile 的 Chrome 进程并重试拉起。
auto-login 触发条件是什么？
当 browser fallback 命中 login / not_logged_in 且环境变量配置了账号密码，会自动登录并重试一次请求。
MCPHub 报 MCP error -32000 怎么排查？
这是通用错误码。需要查看 MCPHub 完整日志定位，常见原因是命令参数、环境变量、网络配置或权限配置错误。
Poke / 公网 endpoint 应该填什么格式？
如果你走 MCPHub，对外地址通常是 你的域名或 IP/mcp/nitan。使用 Tailscale Funnel 时，可尝试 https://xxx.ts.net/login/mcp/nitan。
